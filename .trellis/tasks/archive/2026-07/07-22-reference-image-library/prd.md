# 参考图库与车型素材统一资产模块

## Goal

在 compact-only 工作区新增统一图片资产图库，支持参考图与车型素材双视图、TOS 对象存储、自动 GPT 结构化打标、组合筛选、人工标签覆盖、团队共享和完整图片预览，同时保持现有生成链路不变。

## Requirements

- 新增独立 `/library` 操作台，资产可拥有 `reference`、`vehicle` 一个或两个角色。
- 支持剪贴板、拖放、多文件和文件夹导入；文件夹保留相对层级。
- 图片必须写入已启用的 TOS `public-read` 对象；文件头校验，单图不超过 30 MB。
- 当前用户内按 SHA-256 去重，重复图片直接跳过。
- 导入后自动进入持久化 GPT 打标队列，瞬时错误最多三次，支持失败重试和批量重新打标。
- 标签包括单值图片类型，多值场景/车型/颜色/角度，三态人物字段及自定义标签。
- 人工覆盖按维度锁定；重新打标不得覆盖人工值，空覆盖也必须保留。
- 默认个人可见，可设为团队共享；共享成员只读，所有者与管理员可修改和删除。
- 预览支持筛选结果内前后导航、缩放、平移、缩略图、下载、标签编辑、两级删除及完整键盘操作。
- 管理员可按旧素材文件夹分批复制迁移图片；旧表和现有爆款仿写读取保持不变。
- 本期不接入 GPT-Image-2 生成，只预留资产 ID 选择契约。
- 前台将图片类型、场景、车型、颜色、角度、人物和自由标签统一呈现为 Eagle 式标签块；后台继续保留结构化维度与人工覆盖语义。
- 标签支持自由输入、已有标签自动补全、即时增删、恢复 AI，以及对可编辑资产批量增删；多个统一标签筛选采用 AND。
- `/library` 同步 `professional`、`editorial`、`creator` 三套全局主题；预览舞台保持中性深色，工具栏和详情面板跟随主题。

## Acceptance Criteria

- [ ] PostgreSQL、SQLite 和新安装 schema 均包含图库资产、角色、集合、标签、人工覆盖和打标队列表。
- [ ] 登录用户可导入、分页筛选、编辑和删除自己的资产，并只读查看团队共享资产。
- [ ] 未配置或未启用 TOS 时导入明确失败，不产生本地图库资产。
- [ ] 重复图片返回 `skipped_duplicate`，不新增归属、对象或打标任务。
- [ ] 自动打标状态可观察，失败可重试，人工覆盖在重打标后保持不变。
- [ ] `/library` 在桌面和移动端可完成导入、筛选、批量操作和资产维护。
- [ ] 预览支持按钮、键盘、滚轮/拖拽和触屏操作；删除后索引稳定，焦点返回来源卡片。
- [ ] 移出集合/角色不删除对象；永久删除需确认，并显式显示对象清理失败。
- [ ] 旧素材迁移幂等，非图片、缺失和失败记录可重试；旧生成链路无行为变化。
- [ ] 默认验证不调用真实 TOS、GPT 或图片生成服务，完整 Trellis baseline 通过。
- [ ] 单图和批量统一标签操作即时持久化，团队共享只读资产不能被修改，重新打标保留人工增删结果。
- [ ] 标签自动补全只包含当前账号可见资产，多标签 AND 筛选、键盘 combobox 和移动布局可用。
- [ ] 图库三套主题与全局选择同步并跨页面保持，预览深色舞台和主题化控制面板无重叠或溢出。

## Out Of Scope

- 将新图库接入任何现有内容生成或图片生成流程。
- 迁移非图片文档素材。
- 私有对象 ACL、签名 URL、语义向量检索或人脸识别。
